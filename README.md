Интеллектуальный HR-агент для автоматизации подбора, коммуникации с кандидатами и онбординга новых сотрудников.

Проект реализован как Telegram-бот с поддержкой email-сценариев. Он помогает HR-специалисту принимать и анализировать резюме, вести базу кандидатов, подбирать вакансии, создавать тестовые задания, проводить опросы, назначать собеседования, отправлять офферы и сопровождать сотрудника после выхода на работу.

## Основные Возможности

### 📄 Кандидаты и резюме

- Приём резюме в формате PDF/DOCX через Telegram.
- Приём резюме с почты через IMAP.
- Автоматический парсинг ФИО, контактов, должности, опыта, навыков и другой информации.
- Сохранение кандидатов в PostgreSQL.
- Поиск кандидатов по должности, навыкам и опыту.
- Просмотр карточек кандидатов, редактирование, удаление, восстановление и архивация.
- Безопасное удаление кандидата из архива навсегда и очистка архива кандидатов через подтверждение.
- Экспорт карточки кандидата в PDF и выгрузка списка кандидатов в Excel.

### ✉️ Email и Telegram-коммуникация

Система учитывает канал, через который кандидат отправил резюме:

- если резюме отправлено в Telegram, приглашения и офферы отправляются в Telegram;
- если резюме отправлено на почту, дальнейшие приглашения и офферы отправляются на email;
- в email-сообщениях кандидат получает несколько вариантов связи: перейти в Telegram-бота, ответить письмом или позвонить HR.

Такой подход делает коммуникацию более естественной и не заставляет кандидата резко менять канал взаимодействия.

### 💼 Вакансии и matching

- Создание и редактирование вакансий.
- Хранение требований к вакансии: должность, навыки, опыт, описание.
- Подбор подходящих кандидатов под вакансию.
- Расчёт процента совпадения по навыкам, опыту и должности.
- Архивация вакансий, просмотр архива вакансий и экспорт карточки вакансии в PDF.

### 📝 Тестовые задания

- Генерация тестовых заданий для разных направлений: IT, производство, строительство, логистика, офис, продажи, маркетинг, финансы, HR и другие.
- Для технических направлений можно учитывать уровень Junior/Middle/Senior.
- Есть fallback-шаблоны на случай, если LLM недоступна.
- Последнее тестовое задание можно экспортировать в PDF.

### 📅 Собеседования

- HR может создавать интерактивные слоты для собеседований.
- Кандидат выбирает удобное время из доступных вариантов.
- Слоты можно просматривать, удалять и очищать по датам.
- Поддерживается приглашение кандидата на собеседование по имени или ID.
- При отмене слота кандидат получает уведомление.

### 🎉 Офферы и найм

- HR отправляет кандидату предложение о работе с должностью и зарплатой.
- Кандидат принимает или отклоняет оффер.
- После принятия бот предлагает выбрать дату выхода.
- После выбора даты кандидат получает финальное сообщение, а его интерфейс переключается в режим онбординга.

### 🧭 Онбординг

Онбординг реализован как пошаговый и понятный сценарий адаптации нового сотрудника.

- После принятия оффера бот показывает первую задачу.
- Основное меню меняется на режим онбординга.
- Кандидат может открыть текущую задачу, отметить её выполненной, посмотреть все задачи, все встречи, прогресс и помощь.
- Подсказки по задачам учитывают отдел сотрудника: разработка, аналитика, менеджмент, маркетинг, продажи, HR, бухгалтерия, юридический отдел, дизайн.
- Встроенная база знаний отвечает на частые вопросы: график, столовая, парковка, техника, доступы, наставник, дресс-код, зарплата, отпуск и другие темы.
- Новичок может получить большую часть информации прямо в чате, не тревожа HR по каждому небольшому вопросу.

### 📊 Опросы и аналитика

- Создание NPS-опросов и Pulse-опросов.
- Генерация ссылки для прохождения опроса.
- Сохранение ответов в базе данных.
- Просмотр статистики по результатам.
- Построение графиков по опросам.
- Удаление отдельных опросов или очистка всех опросов.

### 📈 Статистика и экспорт

- Общая статистика по кандидатам, вакансиям и опросам.
- Экспорт кандидатов в Excel.
- Экспорт кандидатов, вакансий, тестовых заданий и планов онбординга в PDF.

## Актуальность и Особенности Проекта

Проект закрывает полный путь кандидата: отправка резюме, обработка данных, приглашение на собеседование, выбор слота, получение оффера, выбор даты выхода и прохождение онбординга.

В отличие от многих HR-систем, агент не ограничивается хранением данных. Он сопровождает кандидата и нового сотрудника в удобном диалоговом формате, а HR-специалист получает меньше ручной работы и меньше повторяющихся вопросов.

Отдельно изучаются способы интеграции с российской платформой MAX. Постепенно планируется добавление функциональности HR-агента в MAX, чтобы расширить каналы взаимодействия и адаптировать систему к отечественным цифровым сервисам.

## Архитектура

```text
Пользователи
HR / кандидаты / новые сотрудники
        |
        v
Telegram Bot + Email
handlers, commands, interview_flow, offer_flow, onboarding_flow
        |
        v
LLM Agent
HRAgent, function calling, ReAct-like tool loop
        |
        v
Tools
parse_resume, search_candidates, match_candidates_to_job,
prepare_interview_invite, generate_test_task и другие
        |
        v
HRAgentFacade
единая точка входа для бизнес-операций
        |
        v
Services
candidate_service, job_matcher, analytics_service,
test_generator, onboarding_generator
        |
        v
Database Layer
candidate_db, jobs_db, survey_db, interview_db, conversation_db
        |
        v

```

Архитектура соответствует агентной LLM-системе:

1. модуль обработки запроса пользователя принимает сообщения, документы и нажатия кнопок;
2. языковая модель анализирует свободные запросы и выбирает действия;
3. система инструментов выполняет операции во внешней среде через function calling;
4. механизм памяти хранит историю взаимодействия и текущий контекст пользователя;
5. модуль формирования ответа возвращает результат в Telegram.

### LLM, Function Calling и ReAct-подобный цикл

Свободные запросы HR передаются в `HRAgent`. Агент получает системный промпт, последние сообщения из истории и список доступных tools. Далее модель может выбрать инструмент автоматически через `tool_choice="auto"`.

В проекте используется не ручной формат `Thought / Action / Observation`, а современный ReAct-подобный цикл на основе function calling:

```text
User request
     |
     v
LLM reasoning
     |
     v
Tool call / Action
     |
     v
Tool result / Observation
     |
     v
Final answer or next tool call
```

Такой подход делает вызовы инструментов структурированными, тестируемыми и безопасными: модель выбирает действие, а фактическое выполнение остаётся под контролем кода.

### Контекст и Память

В системе используется несколько уровней контекста:

- `context.user_data` — краткосрочная память Telegram-сессии: текущий шаг сценария, временные данные, ожидающие подтверждения действия, последний найденный кандидат и последняя вакансия.
- `conversation_history` — долговременная история диалога в PostgreSQL. Она используется для передачи последних сообщений в LLM.
- Бизнес-данные в PostgreSQL — кандидаты, вакансии, архивы, собеседования, опросы и прогресс онбординга.

Благодаря этому поддерживаются более естественные диалоги:

```text
найди инженера-конструктора
пригласи его
```

```text
вакансия 2
создай тест для этой вакансии
```

Критичные действия, например удаление и очистка данных, выполняются только после подтверждения HR.

## Математическая Модель

В проекте используется несколько простых математических моделей, которые помогают формализовать работу HR-агента: оценку соответствия кандидата вакансии, анализ опросов и расчёт прогресса онбординга.

### 🎯 Matching кандидата и вакансии

Итоговая оценка соответствия кандидата вакансии рассчитывается как взвешенная сумма нескольких факторов:

```text
Score = w_skills * S_skills + w_exp * S_exp + w_position * S_position
```

где:

- `Score` — итоговый процент соответствия кандидата вакансии;
- `S_skills` — совпадение навыков кандидата с требованиями вакансии;
- `S_exp` — соответствие опыта работы;
- `S_position` — близость должности кандидата к названию вакансии;
- `w_skills`, `w_exp`, `w_position` — веса факторов.

По умолчанию используются следующие веса:

```text
w_skills = 0.5
w_exp = 0.3
w_position = 0.2
```

Для оценки навыков используется отношение количества найденных совпадений к количеству требуемых навыков:

```text
S_skills = matched_skills / required_skills
```

Оценка опыта нормализуется относительно требуемого опыта:

```text
S_exp = min(candidate_experience / required_experience, 1)
```

Если требуемый опыт равен нулю, значение `S_exp` принимается равным `1`, так как кандидат не должен штрафоваться за отсутствие ограничения по опыту.

### 📌 NPS-опросы

Для оценки лояльности используется показатель NPS:

```text
NPS = %Promoters - %Detractors
```

где:

- `Promoters` — респонденты с оценкой 9-10;
- `Passives` — респонденты с оценкой 7-8;
- `Detractors` — респонденты с оценкой 0-6.

Процент каждой группы рассчитывается по формуле:

```text
GroupPercent = group_count / total_responses * 100
```

### 💓 Pulse-опросы

Для Pulse-опросов рассчитывается средняя оценка по ответам сотрудников:

```text
AverageScore = sum(scores) / count(scores)
```

Эта модель позволяет отслеживать общее состояние сотрудников и динамику удовлетворённости по результатам регулярных опросов.

### ✅ Прогресс онбординга

Прогресс адаптации нового сотрудника рассчитывается как отношение выполненных задач к общему количеству задач:

```text
Progress = completed_tasks / total_tasks * 100
```

Эта формула используется для отображения процента выполнения онбординга и визуального прогресс-бара в чате.

```bash
git clone https://github.com/doSshikK/hr-agent.git
cd hr-agent
```

### 2. Виртуальное окружение

```bash
python3 -m venv venv
source venv/bin/activate
```

Для Windows:

```bash
python -m venv venv
venv\Scripts\activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. PostgreSQL

Создайте базу данных и пользователя:

```sql
CREATE DATABASE hr_agent;
CREATE USER hr_agent WITH PASSWORD 'your_strong_password';
GRANT ALL PRIVILEGES ON DATABASE hr_agent TO hr_agent;
```

Создайте файл `.env` в корне проекта.

```env

LLM_TIMEOUT=180
LLM_MAX_RETRIES=3
```

- `HR_TELEGRAM_ID` можно узнать через `@userinfobot`;
- `OPENROUTER_API_KEY` создаётся в личном кабинете OpenRouter.

### 👤 Сценарий Кандидата

1. Кандидат отправляет резюме в Telegram или на email.
2. Система парсит резюме и сохраняет кандидата.
3. HR приглашает кандидата на собеседование.
4. Кандидат выбирает удобный слот.
5. HR отправляет оффер.
6. Кандидат принимает оффер и выбирает дату выхода.
7. Бот переводит кандидата в режим онбординга.
8. Новый сотрудник проходит задачи, смотрит встречи, прогресс и справку в чате.

## Онбординг в Чате

После начала онбординга кандидат видит отдельное меню:

- Текущая задача.
- Выполнить текущую.
- Помощь по задаче.
- Все задачи.
- Все встречи.
- Прогресс.
- Помощь.

Подсказки по задачам не одинаковые для всех. Они учитывают отдел сотрудника и содержание задачи. Например, для бухгалтерии подсказки связаны с 1С, ЭДО, первичными документами и отчётностью, для HR — с ATS, вакансиями и кандидатами, для дизайна — с Figma, брендбуком и дизайн-системой.

## Опросы

Поддерживаются два типа опросов:

- NPS — оценка лояльности.
- Pulse — оценка состояния и удовлетворённости сотрудников.

После создания опроса HR может отправить ссылку на прохождение. Ответы сохраняются в базе, после чего можно посмотреть статистику и построить график.

## Особенности Реализации

- `HRAgentFacade` используется как единая точка входа к бизнес-логике.
- SQL-операции вынесены в отдельные модули `app/db`.
- Telegram-сценарии разделены по смысловым flow: собеседования, офферы, онбординг, опросы.
- Email-приём и email-отправка вынесены в отдельные утилиты.
- Онбординг отделён от генерации плана: генератор создаёт задачи и встречи, а bot-flow отвечает за интерактивное прохождение.
- Для LLM-генерации предусмотрены fallback-шаблоны.
- LLM использует function calling и ReAct-подобный цикл tool-use для многошаговых запросов.
- История взаимодействия сохраняется в PostgreSQL и используется как контекст для последующих LLM-запросов.
- Для действий с внешним эффектом используется подтверждение HR: например, приглашение на собеседование готовится отдельно и отправляется только после подтверждения.

## API и Инструменты

LLM-агент использует набор внутренних инструментов для выполнения HR-операций:

| Инструмент | Назначение |
| --- | --- |
| `parse_resume` | Извлечение данных из PDF/DOCX-резюме |
| `save_candidate_to_db` | Сохранение или обновление кандидата |
| `search_candidates` | Поиск кандидатов по навыкам, опыту и должности |
| `match_candidates_to_job` | Подбор кандидатов под вакансию |
| `prepare_interview_invite` | Подготовка приглашения на собеседование с подтверждением HR |
| `generate_test_task` | Генерация тестового задания |
| `create_onboarding_plan` | Создание плана онбординга |
| `create_survey` | Создание NPS/Pulse-опроса |
| `analyze_survey` | Анализ результатов опроса |
| `list_jobs` | Получение списка вакансий |
| `list_candidates` | Получение списка кандидатов |
| `get_statistics` | Общая статистика по системе |
